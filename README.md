# Driving & Vehicle Licensing Department (DVLD) Simulation 🚗

## 📖 Project Overview
This project is a comprehensive **simulation** of a Driving and Vehicle Licensing System. It is designed to manage the complex processes of issuing, renewing, and managing driver licenses and vehicle-related services.

---

## 🎯 Project Goal
The primary objective of this project was to practice building a large-scale system with interconnected logic, focusing on:
* **Complex Business Logic:** Handling multi-step processes like applications, tests, and license issuance.
* **Structural Integrity:** Implementing a clean **3-Tier Architecture** to manage data flow effectively.
* **CRUD Mastery:** Mastering Create, Read, Update, and Delete operations across multiple related tables in SQL Server.

---

## 🛠 Development Lifecycle & My Role
I managed the full development cycle of this simulation, focusing on foundational engineering:

1.  **System Analysis:** Analyzing the workflow of license applications, including vision tests, written tests, and practical driving tests.
2.  **Database Design:** Designing a robust relational database on **SQL Server** with multiple relationships (One-to-Many and Many-to-Many) to ensure data consistency.
3.  **Tiered Implementation:** Building the system from scratch using the **3-Tier Architecture** pattern to separate the User Interface from the Data Access logic.

---

## 💻 Technical Stack
* **Architecture:** **3-Tier Architecture** (Presentation, Business Logic, and Data Access Layers).
* **Programming Language:** C# (.NET Framework).
* **Database:** Microsoft SQL Server (Direct SQL queries for deep logic understanding).
* **UI Design:** Windows Forms with a focus on organized user navigation.

---

## ✨ Key Features
* **Application Management:** Handling different types of license applications (New, Renew, Replace).
* **Test Appointments:** Managing the scheduling and results of Vision, Theory, and Practical tests.
* **License Issuance:** Logic-based issuance of licenses upon passing all required tests.
* **People & Users Management:** A centralized system to manage personal information and system users with specific permissions.
* **History Tracking:** Keeping a record of all licenses and applications for every driver.

---

## 🚀 How to Use
1.  **Database:** Restore the provided database backup in your **SQL Server**.
2.  **Connection:** Update the connection string in the **Data Access Layer** to match your local server environment.
3.  **Run:** Open the solution in **Visual Studio** and run the project.
